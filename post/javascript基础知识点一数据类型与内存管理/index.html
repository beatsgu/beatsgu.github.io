JavaScript基础知识点（一）：数据类型与内存管理

1. 前言

理解 JavaScript 中的数据类型和内存管理是掌握这门语言的基础。本文将深入探讨原始类型与对象类型的区别，以及它们在内存中的不同存储方式。

CSDN文章链接

2. 数据类型的内存存储机制

在 JavaScript 中，数据类型分为两大类：原始类型和对象类型，它们有着截然不同的存储和访问方式。

2.1 原始类型：值存储

原始类型包括：number、string、boolean、undefined、null、symbol、bigint

特点：存储的是实际的值，直接保存在栈内存中。

let a = 10;
let b = a;  // b 得到的是 a 的值的副本
b = 20;
console.log(a); // 10 (a 不会被影响)
console.log(b); // 20

2.2 对象类型：引用存储

对象类型包括：Object、Array、Function、Date、RegExp 等

特点：实际的值存储在堆内存中，变量中保存的是指向该堆内存地址的指针（引用）。

let obj1 = { name: "张三" };
let obj2 = obj1;  // obj2 得到的是 obj1 的引用（指针）
obj2.name = "李四";
console.log(obj1.name); // "李四" (obj1 也被修改了)
console.log(obj2.name); // "李四"

2.3 内存模型详解

对于上面的例子，内存分配如下：

  • 变量 obj1 存储了内存地址 #001
  • 在地址 #001 的位置存放了值 { name: "张三" }
  • 当执行 obj2 = obj1 时，复制的是地址 #001
  • 两个变量都指向同一块内存空间，修改任一变量都会影响另一个

3. 经典案例：函数参数传递

让我们通过一个经典案例来加深理解：

function test(person) {
  person.age = 26;  // 修改对象属性
  person = {        // 重新赋值
    name: 'yyy',
    age: 30
  };
  return person;
}

const p1 = {
  name: 'yck',
  age: 25
};

const p2 = test(p1);
console.log(p1); // ?
console.log(p2); // ?

你认为以上代码打印的结果会是什么？

结果：

// 运行结果：
console.log(p1); // { name: 'yck', age: 26 }
console.log(p2); // { name: 'yyy', age: 30 }

3.1 执行过程分析

  1. 初始状态：p1 指向堆内存中的对象 A：{ name: 'yck', age: 25 }

  2. 函数调用 test(p1)：

    function test(person) {
  // 此时 person 参数得到 p1 的引用副本，都指向对象 A

  person.age = 26
  // 通过 person 修改对象 A，p1 也会受影响
  // 对象 A 变为 { name: 'yck', age: 26 }

  person = {
    name: 'yyy',
    age: 30
  }
  // 重新赋值！person 现在指向新的对象 B
  // 但 p1 仍然指向对象 A

  return person  // 返回指向对象 B 的引用
}

  3. 关键概念：重新赋值 ≠ 修改对象

  4. 就是在 test方法中person = { name: 'yyy', age: 30 } 这一行的关键， 那就是赋值操作的本质：重新赋值 ≠ 修改对象

    // 之前的操作
person.age = 26    // 这是修改对象的属性

// 关键的这一行
person = {         // 这是重新赋值，改变引用指向
  name: 'yyy',
  age: 30
}

    重新赋值只是让变量指向新的内存地址，原来的对象依然存在，其他指向它的变量不受任何影响！

3.2 对象比较规则 

// 原始类型比较值
console.log(1 === 1);        // true
console.log("a" === "a");    // true

// 对象类型比较引用地址
console.log({} === {});      // false (不同的内存空间)
console.log([] === []);      // false (不同的内存空间)

let obj = {};
let ref = obj;
console.log(obj === ref);    // true (相同引用)

4. 类型判断方法详解

4.1 typeof 的能力与局限

typeof 对于原始类型基本准确，但对象类型有限制：

// 原始类型判断 - 基本准确
typeof 1;              // "number"
typeof "hello";        // "string"
typeof true;           // "boolean"
typeof undefined;      // "undefined"
typeof Symbol();       // "symbol"
typeof 10n;            // "bigint"

// 特殊情况
typeof null;           // "object" (历史遗留bug)

// 对象类型判断 - 有限制
typeof [];             // "object" (无法区分数组)
typeof {};             // "object"
typeof new Date();     // "object"
typeof /regex/;        // "object"
typeof function() {}; // "function" (函数是特例)

typeof null 的历史问题

​ 对于 null 来说，很多人会认为他是个对象类型，其实这是错误的。虽然 typeof null 会输出 object，但是这只是 JS 存在的一个悠久 Bug。在 JS 的最初版本中使用的是 32 位系统，为了性能考虑使用低位存储变量的类型信息，000 开头代表是对象，然而 null 表示为全零，所以将它错误的判断为 object 。虽然现在的内部类型判断代码已经改变了，但是对于这个 Bug 却是一直流传下来。

4.2 instanceof 原理与应用

instanceof 通过检查原型链来判断对象类型：

// 基本用法
const arr = [];
console.log(arr instanceof Array);   // true
console.log(arr instanceof Object);  // true

const Person = function() {};
const p1 = new Person();
console.log(p1 instanceof Person);   // true

// 注意区别
var str = 'hello world';
console.log(str instanceof String);  // false (原始字符串)

var strObj = new String('hello world');
console.log(strObj instanceof String); // true (String 对象)

原型链查找过程

function Person(name) {
  this.name = name;
}

const p = new Person("张三");

// p instanceof Person 的查找过程：
// 1. 获取 p 的原型链：p.__proto__ -> Person.prototype -> Object.prototype -> null
// 2. 检查 Person.prototype 是否在这个链上
// 3. 返回 true

4.3 最精确的类型判断

使用 Object.prototype.toString 获取精确类型：

function getType(obj) {
  return Object.prototype.toString.call(obj).slice(8, -1).toLowerCase();
}

console.log(getType([]));        // "array"
console.log(getType({}));        // "object"
console.log(getType(null));      // "null"
console.log(getType(new Date())); // "date"
console.log(getType(/regex/));   // "regexp"
console.log(getType(function() {})); // "function"

类型判断方法总结

方法 适用场景 优点 缺点
typeof 原始类型判断 简单快速 对象类型判断有限，null 判断错误
instanceof 对象类型判断 能判断具体对象类型 无法判断原始类型，跨框架可能失效
Object.prototype.toString 精确类型判断 最准确，支持所有类型 语法相对复杂

5. string vs String：原始类型与对象的区别

这是一个经常被忽视但很重要的概念：

5.1 string（原始类型） 

// 创建原始字符串
let str1 = "hello";           // 字面量方式
let str2 = 'world';           // 字面量方式  
let str3 = `template`;        // 模板字符串

console.log(typeof str1);               // "string"
console.log(str1 instanceof String);   // false

5.2 String（构造函数/对象） 

// 创建 String 对象
let strObj1 = new String("hello");  // 对象方式
let strObj2 = String("world");      // 函数调用方式（返回原始字符串）

console.log(typeof strObj1);              // "object"
console.log(strObj1 instanceof String);  // true
console.log(typeof strObj2);              // "string"

5.3 三个关键区别

5.3.1 存储方式 

let primitive = "hello";           // 存储在栈中
let objectStr = new String("hello"); // 引用在栈中，值在堆中

5.3.2 比较行为 

let str1 = "hello";
let str2 = "hello";
let strObj1 = new String("hello");
let strObj2 = new String("hello");

console.log(str1 === str2);     // true (值比较)
console.log(strObj1 === strObj2); // false (引用比较)
console.log(str1 === strObj1);   // false (类型不同)
console.log(str1 == strObj1);    // true (类型转换后比较)

5.3.3 性能差异 

//推荐：原始字符串 - 性能更好
let primitive = "hello";

//不推荐：String 对象 - 性能较差
let objectStr = new String("hello");

实践建议

  1. 优先使用原始类型："hello" 而不是 new String("hello")
  2. 理解引用传递：修改对象参数会影响原始对象
  3. 选择合适的类型判断方法：
    • 基础类型用 typeof
    • 对象类型用 instanceof
    • 需要精确判断用 Object.prototype.toString

6. 小结

本文介绍了 JavaScript 中数据类型的核心概念：原始类型与对象类型的内存存储差异、函数参数传递机制、以及各种类型判断方法的适用场景。
