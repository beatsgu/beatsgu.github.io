JavaScript基础知识点（二）：null、undefined、比较操作符与this讲解

1. 前言

在 JavaScript 基础知识点（一）中，我们了解了原始类型和对象类型（也称为基本数据类型和引用数据类型）。本文将深入探讨其中最容易混淆的概念：null 与 undefined 的区别、比较操作符的差异，以及 JavaScript 中最复杂的 this 指向问题。

CSDN链接

2. null 与 undefined 详解

首先明确一点：Undefined 和 Null 都是基本数据类型，这两个基本数据类型分别都只有一个值，就是 undefined 和 null。

2.1 undefined：表示"未定义"

  • 含义：表示"未定义"或"缺少值"
  • 类型：undefined 类型
  • 产生场景：变量声明但未赋值、函数没有返回值、访问不存在的属性等
  • 特点：通常由 JavaScript 引擎自动产生

2.2 null：表示"空值"

  • 含义：表示"空"或"无"，是一个明确的"空值"
  • 类型：object 类型（这是一个历史bug，在第一篇中有详细讲解）
  • 使用场景：开发者主动赋值，表示变量应该为空
  • 特点：需要开发者主动设置

2.3 基本对比 

// 基本类型检测
console.log(typeof undefined); // "undefined"
console.log(typeof null);      // "object" (历史bug)

// 值比较
console.log(undefined == null);  // true (会进行类型转换)
console.log(undefined === null); // false (严格相等，类型不同)

2.4 产生 undefined 的常见场景 

// 1. 声明变量但未赋值
let name;
console.log(name); // undefined

// 2. 函数参数未传递
function greet(name) {
  console.log(name); // undefined
}
greet();

// 3. 函数没有返回值
function test() {
  // 没有return语句
}
console.log(test()); // undefined

// 4. 访问不存在的对象属性
let person = { name: "张三" };
console.log(person.age); // undefined

// 5. 访问不存在的数组元素
let arr = [1, 2, 3];
console.log(arr[10]); // undefined

2.5 null 的使用场景 

// 1. 主动设置为空值
let data = null; // 明确表示这个变量目前为空

// 2. 清空对象引用
let obj = { name: "张三" };
obj = null; // 断开引用，帮助垃圾回收

// 3. API返回空值
// 很多API在没有数据时返回null
let result = document.getElementById("nonexistent"); // null

// 4. 初始化变量（当你知道后续会赋值对象时）
let user = null; // 稍后会赋值为用户对象

2.6 语义上的区别 

// undefined - "这里应该有值，但目前没有定义"
let userAge; // 用户年龄还没获取到

// null - "这里明确地没有值"
let userAvatar = null; // 用户明确没有头像

2.7 常见陷阱 

// 陷阱1：typeof null
console.log(typeof null); // "object" 而不是 "null"，历史遗留bug

// 陷阱2：数组中的空元素
let arr = [1, , 3]; // 第二个元素是 undefined
console.log(arr[1]); // undefined
console.log(arr.length); // 3

// 陷阱3：解构赋值
let { name, age } = {}; 
console.log(name); // undefined (不是 null)

// 陷阱4：函数参数
function test(a, b) {
  console.log(a, b); // undefined undefined
}
test(); // 没传参数

2.8 对比总结

特性 undefined null
含义 未定义、缺少值 明确的空值
类型 undefined object (历史 bug)
产生方式 系统自动产生 开发者主动设置
JSON 序列化 被忽略 保留为 null
默认参数 触发默认值 不触发默认值
使用建议 表示 “还未定义” 表示 “明确为空”

通俗理解：

  • undefined = “我不知道这里有什么”
  • null = “我知道这里是空的”

3. 比较操作符详解：==、=== 与 Object.is()

在 JavaScript 中进行判断时，有三种主要的比较方式：双等号（==）、三等号（===）和 Object.is()，它们各有特点。

3.1 核心区别概述

  • 双等号（==）：使用双等号（==）进行相等判断时，如果两边的类型不一致，则会进行强制类型转化后再进行比较。
  • 三等号（===）：使用三等号（===）进行相等判断时，如果两边的类型不一致时，不会做强制类型准换，直接返回 false。
  • Object.is()：Object.is()是ES6新增方法, 使用 Object.is 来进行相等判断时，一般情况下和三等号的判断相同，但它处理了一些特殊的情况，比如 -0 和 +0 不再相等，两个 NaN是相等的。

3.2 基本类型比较 

// 数字比较
console.log(1 == 1);          // true
console.log(1 === 1);         // true
console.log(Object.is(1, 1)); // true

// 字符串比较
console.log("hello" == "hello");          // true
console.log("hello" === "hello");         // true
console.log(Object.is("hello", "hello")); // true

3.3 类型转换差异 

// == 比较时,会先进行类型转换,再比较
console.log(1 == "1");          // true (字符串转为数字)
console.log(true == 1);         // true (布尔值转为数字,true → 1，false → 0)

console.log(null == undefined); // true (特殊规则)
console.log("" == 0);           // true (空字符串转为0)

// === 和 Object.is() 不进行类型转换
console.log(1 === "1");          // false
console.log(true === 1);         // false
console.log(null === undefined); // false
console.log("" === 0);           // false

console.log(Object.is(1, "1"));          // false
console.log(Object.is(true, 1));         // false
console.log(Object.is(null, undefined)); // false
console.log(Object.is("", 0));           // false

3.4 布尔值转换的特殊规则 

// 布尔值会先转为数字：true → 1，false → 0
console.log(true == 1);  // true
console.log(false == 0); // true

// 但是其他数字对比会返回false
console.log(true == 2);  // false
console.log(false == -1); // false

3.5 特殊值的处理差异

这是三者最重要的区别所在：

3.5.1 NaN 的比较 

console.log(NaN == NaN);          // false
console.log(NaN === NaN);         // false
console.log(Object.is(NaN, NaN)); // true

3.5.2 +0 和 -0 的比较 

console.log(+0 == -0);          // true
console.log(+0 === -0);         // true
console.log(Object.is(+0, -0)); // false

3.6 常见"坑"场景

这种隐式转换可能导致反直觉的结果：

console.log(true == "1");  // true（"1"转为数字1，true转为1）
console.log(false == 0);   // true（false转为0）
console.log(false == "");  // true（""转为数字0，false转为0）
console.log("0" == false); // true（false转为0，"0"转为0）

3.7 使用建议

  • 一般情况：推荐使用 ===，避免意外的类型转换
  • 特殊需求：当需要精确区分 NaN 或 ±0 时，使用 Object.is()
  • 避免使用：除非明确需要类型转换，否则避免使用 ==

4. this 指向讲解

4.1 对 this 对象的理解

this 是JavaScript中的一个关键字，它是执行上下文中的一个属性，指向最后一次调用这个方法的对象。

核心原则

  • this 在运行时绑定，不是编写时绑定
  • this 的值由函数如何被调用决定
  • 每次函数调用都会重新确定 this 的值

4.2 场景讲解 

// 不管 foo 函数在哪里定义，直接调用时 this 都指向全局对象
function foo() {
  console.log(this.a) // this === window (浏览器) 或 global (Node.js)
}

var a = 1 // 全局变量
// 第一种**函数调用模式**    (默认绑定)
foo() // 1

// 第二种**方法调用模式**    ( 隐式绑定)
const obj = {
  a: 2,
  foo: foo
}
obj.foo()

// 第三种**构造器调用模式**   (new 绑定)
const c = new foo()

以上场景分析:

1.对于直接调用 foo 来说，不管 foo 函数被放在了什么地方，this 一定是 window 1

2.对于 obj.foo() 来说，我们只需要记住，谁调用了函数，谁就是 this，所以在这个场景下 foo 函数中的 this 就是 obj 对象2

3.对于 new 的方式来说，this 被永远绑定在了 c 上面，不会被任何方式改变 this 3

以上是this的三种绑定方式,还有第四种绑定方式:

显式绑定（call、apply、bind）

const person1 = { name: "张三" };
const person2 = { name: "李四" };

function introduce(age, city) {
  console.log(`我是 ${this.name}，${age}岁，来自${city}`);
}

// call：立即调用，参数逐个传递
introduce.call(person1, 25, "北京");    // "我是 张三，25岁，来自北京"
introduce.call(person2, 30, "上海");    // "我是 李四，30岁，来自上海"

// apply：立即调用，参数以数组形式传递
introduce.apply(person1, [25, "北京"]); // "我是 张三，25岁，来自北京"

// bind：返回新函数，不立即调用
const introduceZhangSan = introduce.bind(person1, 25);
introduceZhangSan("北京"); // "我是 张三，25岁，来自北京"

apply 、 call 和 bind 调用模式，这三个方法都可以显示的指定调用函数的 this 指向。4

总结: this 的指向可以通过四种调用模式来判断。

优先级: 这四种方式，使用构造器调用模式的优先级最高，然后是 apply、call和 bind 调用模式，然后是方法调用模式，然后是函数调用模式。

4.3 箭头函数的 this

核心特点

箭头函数并没有属于⾃⼰的 this，它所谓的 this 是捕获其所在上下⽂的 this 值(也可以说是继承外层作用域的this)，作为⾃⼰的 this 值，并且由于没有属于⾃⼰的 this，所以是不会被 new调⽤的，这个所谓的 this 也不会被改变。

4.3.1 场景1 

const obj = {
  name: "张三",
  
  // 普通函数：this 由调用方式决定
  normalFunction() {
    console.log("普通函数:", this.name);
    
    // 箭头函数：继承外层作用域的 this（即 obj）
    const arrowFunction = () => {
      console.log("箭头函数:", this.name);
    };
    
    arrowFunction();
  }
};

obj.normalFunction();
// 输出：
// 普通函数: 张三
// 箭头函数: 张三

4.3.2 场景2: 嵌套箭头函数 

function a() {
  return () => {
    return () => {
      console.log(this);
    };
  };
}

console.log(a()()()); // this 指向 window
// 因为包裹箭头函数的第一个普通函数是 a，所以 this 是 window

4.3.3 箭头函数的 this 无法改变 

const obj = { name: "张三" };

const arrow = () => {
  console.log(this.name);
};

// 箭头函数的 this 无法通过 call、apply、bind 改变
arrow.call(obj);   // undefined (this 仍指向全局对象)
arrow.apply(obj);  // undefined
arrow.bind(obj)(); // undefined

// 对比普通函数
function normal() {
  console.log(this.name);
}

normal.call(obj); // "张三"

4.4 实际应用场景

4.4.1 事件处理中的 this 

const button = document.querySelector('button');
const obj = {
  name: "按钮对象",
  
  // 普通函数：this 指向 button 元素
  handleClick: function() {
    console.log(this); // button 元素
  },
  
  // 箭头函数：this 指向 obj
  handleClickArrow: () => {
    console.log(this); // obj 对象
  }
};

button.addEventListener('click', obj.handleClick);
button.addEventListener('click', obj.handleClickArrow);

4.4.2 定时器中的 this 

const obj = {
  name: "定时器对象",
  
  start: function() {
    // 普通函数：this 指向 window
    setTimeout(function() {
      console.log(this.name); // undefined
    }, 1000);
    
    // 箭头函数：this 指向 obj
    setTimeout(() => {
      console.log(this.name); // "定时器对象"
    }, 1000);
  }
};

obj.start();

5. 总结

5.1 null 和 undefined

  • 使用 undefined 表示"未定义的值"
  • 使用 null 表示"明确的空值"
  • 避免手动设置变量为 undefined

5.2 比较操作符

  • 优先使用 === 进行比较，避免类型转换陷阱
  • 需要精确比较 NaN 或区分 ±0 时使用 Object.is()
  • 谨慎使用 ==，除非明确需要类型转换

5.3 this 指向

  • 记住四种绑定方式和优先级
  • 箭头函数继承外层作用域的 this，无法改变
  • 在需要保持 this 指向的场景下优先考虑箭头函数
  • 使用 call、apply、bind 显式绑定 this

6. 小结

本文深入讲解了 JavaScript 中容易混淆的核心概念：null 与 undefined 的语义差异、三种比较操作符的行为区别，以及复杂但重要的 this 指向机制。

  1. 第一种是函数调用模式，当一个函数不是一个对象的属性时，直接作为函数来调用时，this 指向全局对象。 ↩︎

  2. 第二种是方法调用模式，如果一个函数作为一个对象的方法来调用时，this 指向这个对象。 ↩︎

  3. 第三种是构造器调用模式，如果一个函数用 new 调用时，函数执行前会新创建一个对象，this 指向这个新创建的对象。 ↩︎

  4. 第四种是apply 、 call 和 bind 调用模式，这三个方法都可以显示的指定调用函数的 this 指向。其中 apply 方法接收两个参数：一个是 this 绑定的对象，一个是参数数组。call 方法接收的参数，第一个是 this 绑定的对象，后面的其余参数是传入函数执行的参数。也就是说，在使用 call() 方法时，传递给函数的参数必须逐个列举出来。bind 方法通过传入一个对象，返回一个 this 绑定了传入对象的新函数。这个函数的 this 指向除了使用 new 时会被改变，其他情况下都不会改变。 ↩︎
